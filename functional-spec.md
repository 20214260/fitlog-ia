## 📐 명세 작성 규칙

- **요소 ID 체계**: `{화면ID}-{UI종류}-{번호}` (예: `P07-BTN-01` = 홈 화면의 첫 번째 버튼)
- **기능 ID 체계**: `F{화면번호}-{순번}` (예: `F07-01`)
- **UI 종류 약어**: BTN(버튼), TXT(텍스트), IMG(이미지), INP(입력), CARD(카드), LIST(리스트), TAB(탭), MOD(모달), TST(토스트)
- **모호 표현 금지**: "적절히", "필요시" 등 사용 안 함. 모든 수치·조건은 구체적 명시.

# 🔵 P07 — 홈 (대시보드)

## 화면 개요
| 항목 | 내용 |
|------|------|
| 화면 ID | P07 |
| 화면명 | 홈 (대시보드) |
| 진입 경로 | ① 로그인 성공 후 자동 / ② 하단 탭바 "홈" 탭 / ③ 다른 화면에서 "홈으로" |
| 화면 목적 | 오늘의 루틴 + AI 추천 + 일일 요약 제공, 주요 기능의 허브 |
| 관련 시나리오 | A (신규→첫 운동), B (일상 기록), C (분석) |

## UI 요소 정의

| 요소 ID | 요소명 | 종류 | 위치 | 설명 |
|---------|--------|------|------|------|
| P07-TXT-01 | 인사말 | TXT | 상단 좌 | "안녕하세요, {닉네임}님" (최대 20자, 초과 시 ... 처리) |
| P07-BTN-01 | 알림 아이콘 | BTN | 상단 우 | 미확인 알림 수 뱃지(빨강 원, 1~99, 100 이상은 "99+") |
| P07-CARD-01 | 오늘의 요약 카드 | CARD | 상단 | 운동 시간(분), 소모 칼로리(kcal), 총 볼륨(kg) 3개 지표 |
| P07-CARD-02 | AI 추천 루틴 카드 | CARD | 중단 | 추천 루틴명 + 예상 소요시간 + 난이도(1~5★) |
| P07-BTN-02 | 빠른 시작 버튼 | BTN | CARD-02 하단 | "시작하기" - 추천 루틴 즉시 실행 |
| P07-LIST-01 | 최근 운동 리스트 | LIST | 하단 | 최근 3일간 운동 이력 (최대 5개) |
| P07-TAB-01 | 하단 탭바 | TAB | 최하단 고정 | 홈/운동/AI/통계/마이 5개 탭 |

## 기능 명세 (Trigger → Action)

| 기능 ID | 트리거 (사용자 동작) | 액션 (시스템 반응) | 결과 |
|---------|---------------------|-------------------|------|
| F07-01 | P07-BTN-01 알림 아이콘 탭 | 알림 페이지로 슬라이드 전환 | 알림 화면 진입 |
| F07-02 | P07-CARD-01 오늘 요약 카드 탭 | P16 통계 대시보드로 이동 | 통계 화면 진입 |
| F07-03 | P07-CARD-02 AI 추천 카드 탭 | P14 추천 루틴 상세로 이동 | 루틴 미리보기 |
| F07-04 | P07-BTN-02 "시작하기" 탭 | P11 운동 상세/기록 화면으로 이동, 선택된 routineId 전달 | 즉시 운동 시작 |
| F07-05 | P07-LIST-01 항목 탭 | 해당 일자의 P19 일일 기록 상세로 이동 | 과거 기록 조회 |
| F07-06 | 화면 상단에서 아래로 끌어내림 (Pull-to-refresh) | 1초 로딩 인디케이터 표시 후 데이터 재조회 | 최신 데이터 갱신 |
| F07-07 | P07-TAB-01 탭바 항목 탭 | 해당 탭 화면으로 즉시 전환 (애니메이션 없음) | 탭 전환 |

## 예외/제한 사항

- **데이터 없음 (Empty State)**: 첫 가입 후 운동 기록 0건일 때 → CARD-01 자리에 "첫 운동을 시작해보세요!" 메시지 + 일러스트
- **네트워크 오류**: 데이터 로드 실패 시 → CARD 자리에 회색 스켈레톤 + "다시 시도" 버튼
- **닉네임 길이**: 최대 20자, 21자 이상은 "...님"으로 truncate
- **AI 추천 없을 때**: CARD-02 자리에 "운동 기록을 더 쌓으면 AI 추천을 받을 수 있어요" 안내 (최소 3회 기록 필요)
- **로딩 시간**: API 응답 2초 초과 시 스켈레톤 UI 표시
- **알림 뱃지 상한**: 100 이상은 "99+"로 고정 표시

# 🟠 P09 — 운동 종류 선택

## 화면 개요
| 항목 | 내용 |
|------|------|
| 화면 ID | P09 |
| 화면명 | 운동 종류 선택 |
| 진입 경로 | P08 운동 기록 → "운동 추가" 탭 |
| 화면 목적 | 헬스/유산소/홈트 카테고리에서 운동 종목 선택 |
| 관련 시나리오 | A, B (필수 경로) |

## UI 요소 정의

| 요소 ID | 요소명 | 종류 | 위치 | 설명 |
|---------|--------|------|------|------|
| P09-BTN-01 | 뒤로가기 | BTN | 상단 좌 | 이전 화면(P08)으로 복귀 |
| P09-INP-01 | 검색 입력창 | INP | 상단 | 운동 종목명 검색 (placeholder: "운동 검색") |
| P09-TAB-01 | 카테고리 탭 | TAB | 검색창 아래 | 헬스/유산소/홈트 3개 탭 |
| P09-LIST-01 | 운동 종목 리스트 | LIST | 본문 | 카테고리별 운동 종목 목록 |
| P09-BTN-02 | 즐겨찾기 토글 | BTN | LIST 각 항목 우 | ⭐ 즐겨찾기 ON/OFF |
| P09-BTN-03 | 운동 선택 | BTN | LIST 각 항목 | 종목 선택 → P11 진입 |

## 기능 명세 (Trigger → Action)

| 기능 ID | 트리거 | 액션 | 결과 |
|---------|--------|------|------|
| F09-01 | P09-BTN-01 뒤로가기 탭 | 이전 화면으로 슬라이드 백 | P08 복귀 |
| F09-02 | P09-INP-01에 2자 이상 입력 | 0.3초 debounce 후 자동 검색 시작 | 실시간 검색 결과 노출 |
| F09-03 | P09-TAB-01 탭 항목 선택 | 해당 카테고리의 LIST-01 데이터 갱신 | 카테고리 필터링 |
| F09-04 | P09-BTN-02 즐겨찾기 토글 탭 | 별 아이콘 채움/비움 토글 + 서버 동기화 | 즐겨찾기 상태 변경 |
| F09-05 | P09-BTN-03 운동 종목 탭 | P11 운동 상세/기록 진입, exerciseId 전달 | 운동 기록 시작 |
| F09-06 | 검색 결과 0건 | LIST-01 자리에 빈 상태 화면(E03) 표시 + "직접 추가하기" 버튼 | Unhappy Path 진입 |

## 예외/제한 사항

- **검색어 길이**: 최소 2자 이상부터 검색 (1자 이하는 검색 미실행), 최대 30자
- **검색어 특수문자**: 한글/영문/숫자만 허용, 특수문자 입력 시 무시
- **검색 결과 없음**: "'{검색어}'에 대한 결과가 없습니다" + "직접 추가하기" 버튼 노출
- **카테고리별 종목 수**: 헬스 200+, 유산소 30+, 홈트 80+ 기본 제공
- **즐겨찾기 한도**: 사용자당 최대 50개, 초과 시 toast "즐겨찾기는 최대 50개까지 가능합니다"
- **네트워크 오류 시**: 마지막 캐시된 리스트 사용 + 상단에 "오프라인 모드" 배너

# 🔴 P11 — 운동 상세 / 기록

## 화면 개요
| 항목 | 내용 |
|------|------|
| 화면 ID | P11 |
| 화면명 | 운동 상세 / 기록 |
| 진입 경로 | P09 운동 종목 선택 / P07 빠른 시작 / P14 추천 루틴 시작 |
| 화면 목적 | 세트별 중량·횟수 입력 및 기록 |
| 관련 시나리오 | A, B (핵심 입력 화면) |

## UI 요소 정의

| 요소 ID | 요소명 | 종류 | 위치 | 설명 |
|---------|--------|------|------|------|
| P11-TXT-01 | 운동명 | TXT | 상단 | 선택한 운동 종목명 (최대 30자) |
| P11-BTN-01 | 닫기 | BTN | 상단 우 | X 아이콘, 운동 중단 |

| P11-LIST-01 | 세트 리스트 | LIST | 본문 | 세트 1, 2, 3... 각 행마다 중량·횟수 입력 |
| P11-INP-01 | 중량 입력 | INP | LIST 행 내 | 숫자 키패드, 단위 kg (0.5kg 단위) |
| P11-INP-02 | 횟수 입력 | INP | LIST 행 내 | 숫자 키패드, 단위 회 (정수) |
| P11-BTN-02 | 세트 완료 체크 | BTN | LIST 행 우 | ○ → ✓ 토글, 완료 시 다음 세트 활성화 |
| P11-BTN-03 | 세트 추가 | BTN | LIST 하단 | "+ 세트 추가" |
| P11-BTN-04 | 운동 완료 | BTN | 화면 하단 고정 | "운동 완료" - P12로 이동 |
| P11-TXT-02 | 타이머 | TXT | 상단 우 | 휴식 시간 카운트다운 (M:SS) |

## 기능 명세 (Trigger → Action)

| 기능 ID | 트리거 | 액션 | 결과 |
|---------|--------|------|------|
| F11-01 | P11-BTN-01 닫기 탭 | "운동을 중단하시겠어요?" 확인 모달 표시 | 사용자 확인 후 P09로 복귀 (기록 미저장) |
| F11-02 | P11-INP-01 중량 입력 | 0.5kg 단위 숫자 키패드 노출, 입력값 실시간 반영 | 입력값 저장 |
| F11-03 | P11-INP-02 횟수 입력 | 정수 숫자 키패드 노출, 입력값 실시간 반영 | 입력값 저장 |
| F11-04 | P11-BTN-02 세트 완료 체크 탭 | 해당 행 ✓ 표시 + 60초 휴식 타이머 자동 시작 + 다음 세트 입력 자동 활성화 | 세트 완료 처리 |
| F11-05 | P11-BTN-03 세트 추가 탭 | LIST-01 하단에 새 빈 세트 행 추가 | 세트 추가 |
| F11-06 | P11-BTN-04 운동 완료 탭 | 입력값 검증 → 통과 시 서버 저장 → P12 진입 | 운동 종료 |
| F11-07 | 입력값 0 또는 빈값 상태에서 F11-06 | toast "중량과 횟수를 입력해주세요" 노출 + 미입력 행 빨간 테두리 강조 | 진행 차단 |
| F11-08 | 네트워크 끊김 상태에서 F11-06 | 로컬 임시 저장 + toast "오프라인 모드, 연결 시 자동 저장됩니다" | Unhappy Path 진입 |

## 예외/제한 사항

- **중량 입력 범위**: 0.5kg ~ 999.5kg (0.5kg 단위), 범위 초과 시 toast "0.5~999.5kg 사이로 입력해주세요"
- **횟수 입력 범위**: 1 ~ 999회, 범위 초과 시 toast "1~999회 사이로 입력해주세요"
- **세트 최대 개수**: 운동 1개당 최대 20세트, 초과 시 P11-BTN-03 비활성화 + tooltip "최대 20세트까지 가능합니다"
- **운동명 길이**: 30자 초과 시 ... 처리
- **휴식 타이머**: 기본 60초, 사용자 설정으로 30/60/90/120초 변경 가능
- **자동 저장 주기**: 30초마다 로컬 캐시 자동 저장 (앱 강제 종료 대비)
- **네트워크 오류**: 운동 완료 시점에 오프라인이면 → 로컬 저장 + 재연결 시 자동 동기화 (Unhappy Path)
- **앱 백그라운드 진입 시**: 타이머 백그라운드 실행 + 진동 알림으로 휴식 종료 안내

# 🟢 P12 — 운동 완료 요약

## 화면 개요
| 항목 | 내용 |
|------|------|
| 화면 ID | P12 |
| 화면명 | 운동 완료 요약 |
| 진입 경로 | P11 운동 완료 버튼 |
| 화면 목적 | 방금 완료한 운동의 결과 요약, 다음 행동 유도 |
| 관련 시나리오 | A, B (종료점) |

## UI 요소 정의

| 요소 ID | 요소명 | 종류 | 위치 | 설명 |
|---------|--------|------|------|------|
| P12-IMG-01 | 완료 일러스트 | IMG | 상단 | 축하 애니메이션 (Lottie 1회 재생) |
| P12-TXT-01 | 축하 메시지 | TXT | IMG 하단 | "수고하셨습니다! 🎉" |
| P12-CARD-01 | 결과 요약 카드 | CARD | 중단 | 운동 시간/소모 칼로리/총 볼륨 3개 지표 |
| P12-LIST-01 | 세트 기록 | LIST | 본문 | 각 세트별 중량·횟수 표시 |
| P12-INP-01 | 메모 입력 | INP | 하단 | 운동 일지 자유 입력 (선택) |
| P12-BTN-01 | 공유 | BTN | 하단 좌 | SNS 공유 |
| P12-BTN-02 | 통계 보기 | BTN | 하단 중 | P16 통계 대시보드로 이동 |
| P12-BTN-03 | 홈으로 | BTN | 하단 우 | P07 홈으로 복귀 |

## 기능 명세 (Trigger → Action)

| 기능 ID | 트리거 | 액션 | 결과 |
|---------|--------|------|------|
| F12-01 | P11에서 진입 직후 | Lottie 애니메이션 1회 재생 (2.5초) + 햅틱 피드백 1회 | 진입 효과 |
| F12-02 | P12-INP-01에 메모 입력 후 포커스 아웃 | 자동 저장 (debounce 1초) | 메모 저장 |
| F12-03 | P12-BTN-01 공유 탭 | OS 공유 시트 노출 (이미지 + 텍스트 생성) | 외부 앱 공유 |
| F12-04 | P12-BTN-02 통계 보기 탭 | P16 통계 대시보드로 이동 | 통계 진입 |
| F12-05 | P12-BTN-03 홈으로 탭 | P07 홈으로 이동 (백 스택 초기화) | 시나리오 종료 |
| F12-06 | 진입 시 자동 | 운동 데이터를 서버 저장 → 통계 갱신 트리거 | 데이터 영속화 |

## 예외/제한 사항

- **메모 글자 수**: 최대 500자, 카운터 표시 ("0/500"), 초과 시 입력 차단
- **메모 줄바꿈**: 줄바꿈 최대 10회, 초과 시 입력 차단
- **공유 이미지 생성 실패 시**: toast "이미지 생성에 실패했습니다. 텍스트로 공유합니다"
- **결과 0건 (오류)**: CARD-01에 "기록을 불러올 수 없습니다" + "다시 시도" 버튼 (서버 측 데이터 오류 대비)
- **백버튼 처리**: 안드로이드 백버튼 = P12-BTN-03과 동일 동작 (홈 이동)
- **수치 표시 형식**: 시간 "MM분" / 칼로리 "XXX kcal" / 볼륨 "XXX kg" (천 단위 콤마 없음)
- **수치 상한**: 운동 시간 최대 999분, 칼로리 최대 9999 kcal, 볼륨 최대 99999 kg

- **데이터 없음 (신규 사용자)**: 모든 차트 자리에 "데이터가 쌓이면 통계가 표시됩니다" + "운동 시작하기" 버튼
- **선택 기간 데이터 0건**: 차트 자리에 "이 기간엔 운동 기록이 없어요" 메시지
- **그래프 로딩**: 데이터 로드 중 스켈레톤 차트(회색 막대) 표시
- **숫자 표시**: 천 단위 콤마 표시 (예: "12,500 kg")
- **기간 표시 형식**: 주("5/12~5/18") / 월("2025년 5월") / 년("2025년")
- **차트 데이터 상한**: 한 차트당 최대 데이터 포인트 365개 (년 단위), 초과 시 주 단위로 자동 그룹핑
- **네트워크 오류**: 마지막 캐시 데이터 표시 + "최신 데이터를 불러올 수 없습니다" 배너
