##  사용자 시나리오

페이지는 다음 4가지 시나리오를 기반으로 도출했습니다.

###  시나리오 A: 신규 사용자 온보딩 → 첫 운동 기록
김초보(28세, 헬스 입문)가 앱 다운로드 후 → 회원가입 → 신체 정보 입력 → 운동 목표 설정(체중 감량) → AI 추천 루틴 확인 → 오늘 운동 시작 → 세트별 기록 → 완료 요약 확인

**경로**: P01 → P03 → P04 → P05 → P06 → P07 → P08 → P09 → P11 → P12

###  시나리오 B: 기존 사용자의 일상적 운동 기록 → 통계 확인
박중급(34세, 헬스 2년차)이 출근 전 헬스장에서 → 로그인 → 홈에서 오늘의 루틴 진입 → 벤치프레스 기록(중량/세트/횟수) → 유산소 30분 추가 → 통계에서 주간 볼륨 확인

**경로**: P07 → P08 → P09 → P11 → P12 → P16 → P17

###  시나리오 C: 데이터 분석 → AI 코칭 → 목표 재설정
한 달 후 → 통계 대시보드에서 부위별 볼륨 그래프 확인 → AI 코치가 "하체 비중 부족" 알림 → 추천 하체 루틴 저장 → 마이페이지에서 목표 수정

**경로**: P16 → P17 → P15 → P14 → P22

###  시나리오 D: 예외 흐름
네트워크 끊김(E02) / 검색 결과 없음(E03) / 잘못된 URL 접근(E01) / 데이터 로딩(E04)

##  그룹핑 전략 & 사용자 동선

논리적 그룹핑은 **사용자 행동 흐름**을 기준으로 했습니다.

- **진입 그룹**: 앱 첫 진입 ~ 초기 설정까지의 단방향 흐름
- **메인 그룹**: 모든 핵심 기능의 허브 (홈 화면)
- **운동 그룹**: 매일 반복되는 핵심 사용 동선 (기록의 메인 흐름)
- **AI 그룹**: 추천/분석 같은 부가 기능을 한 곳에 묶어 탐색 부담 감소
- **통계 그룹**: 데이터 회고 영역 - 사용자가 "내 성장 확인" 목적으로 진입
- **사용자 그룹**: 계정·설정처럼 진입 빈도는 낮지만 깊이 있는 영역
- **예외 그룹**: 모든 화면에서 발생 가능한 비정상 상황 처리

### 동선 설계 근거
- 홈(P07)을 모든 핵심 기능의 허브로 설정 → 최대 2탭 안에 주요 기능 도달
- 자주 쓰는 운동 기록(P08~P12)은 홈에서 1탭, 통계는 탭바에서 1탭
- 깊이 있는 설정 영역(P23 → P24/P25)은 Depth 3까지 허용, 일상 사용 흐름은 Depth 2 이내 유지
